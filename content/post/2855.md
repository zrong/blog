我们都知道，腾讯文档的体验有点差。

尤其最近上线的那个 AI 模式，不仅 UI 设计糟糕，交互体验混乱，甚至都无法找到需要的文件。

![腾讯文档中的 AI 助手功能拉跨](/uploads/2026/openclaw-tencent-docs-1.png)

昨天，[腾讯文档官方公众号发布了一篇文章](https://mp.weixin.qq.com/s/snbdk90liD-dHR0sb_n2ow)，介绍了使用 OpenClaw 调用腾讯文档 Skill，今天曾老师就来讲讲如何使用。

本质上，这个 Skill 是通过 MCP 调用腾讯文档的功能的。我这里的范例使用的是 Claude Code，但使用 OpenClaw 或者其他 Agent 也是没问题的。

## 安装 Skill

在 [Skill 是 Agent 的核心：让龙虾 OpenClaw 给飞书发电脑桌面截图]({{< relref "post/2852.md" >}}) 一文中，我提到了曾老师的 Skill 仓库，也介绍了如何安装 Skill，欢迎阅读。

使用下面的命令安装 skill：

```
npx skills add zrong/skills --skill tencent-docs --agent claude-code --agent openclaw
```

当然，也可以直接告诉龙虾 OpenClaw 或者 Claude Code：

> 帮我安装 https://github.com/zrong/skills  中的 tencent-docs

## 配置 MCP

Skill 安装后，问一下是否能用：

![配置 MCP](/uploads/2026/openclaw-tencent-docs-2.png)

再要求操作腾讯文档：

> 帮我操作下腾讯文档
 
此时 Agent 会要求提供 token。

![提供 TOKEN](/uploads/2026/openclaw-tencent-docs-3.png)

在腾讯文档中，「空间」的右上角三道杠菜单获取 MCP token，将其提供给龙虾 OpenClaw。

![在腾讯文档中获取空间的 token](/uploads/2026/openclaw-tencent-docs-4.png)


把得到的 MCP token 大概龙虾就可以了：

![让 Agent 自动配置 MCP](/uploads/2026/openclaw-tencent-docs-5.png)

## 计算 PDF 文档字数

腾讯文档自带的 AI 问答，搜索能力很弱，所以用 Skill 的搜索能力，下面这个问题，它就搞不定：

> 2603制作 这个文件夹中，有 5 个 PDF 文件，你统计一下这些 PDF 文件的字数。注意要写个 Python 脚本来统计，不要乱猜。

需要注意的是，即使文档是 PDF 格式，在腾讯文档内部，也并不是这样定义的。

![搜不到 PDF](/uploads/2026/openclaw-tencent-docs-8.png)

对于腾讯文档来说，它会认为这些 PDF 是 Word 格式。

![文档中全是文档](/uploads/2026/openclaw-tencent-docs-9.png)

找到后，龙虾就开始自己写脚本。

为什么一定要使用 Python 统计呢？因为基于大模型的特点，字数统计功能是不太准的，使用代码才能准确获取数据。

至于如何读取文件，如何检测字数，大模型自然知道调用什么函数，你自己不用操心。

![统计文档字数](/uploads/2026/openclaw-tencent-docs-10.png)
